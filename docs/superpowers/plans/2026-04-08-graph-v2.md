# GraphTrace Graph v2 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn GraphTrace's current bounded graph panel into a graph-first investigation workspace with overview/focused/flow modes, stronger graph controls, and readable self-host layouts.

**Architecture:** Keep `@xyflow/react` as the interaction renderer, but introduce a dedicated graph view-model layer that derives a bounded `view graph` from workspace/repository/package state, graph mode, focus node, and filters. Split the UI into stable graph controls, canvas, and inspector units so search, graph state, and drill-down actions can evolve independently without turning `App.tsx` and `graph-workspace.tsx` into a single monolith.

**Tech Stack:** React, TypeScript, `@xyflow/react`, existing `apps/web` state model, Vitest, Biome, Vite
