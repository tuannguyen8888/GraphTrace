# UI I18n Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add first-class `en`/`vi` locale support to the multi-workspace web UI, with English as default and persisted locale selection.

**Architecture:** Introduce a lightweight in-app translation module instead of pulling a full i18n library. Route state and local storage will carry the selected locale, while UI/view-model helpers will consume a shared translator so home/detail/graph/search surfaces render consistently.

**Tech Stack:** React, TypeScript, existing `apps/web` state model, Vitest
